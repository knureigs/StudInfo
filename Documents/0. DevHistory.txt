Проект для получения данных из рейтингов студентов

Как определить по фамилии и инициалам, это наш студент, или нет? Это частая задача при подготовке отчетов по НИРСу, когда для каждого соавтора поданой публикации нужно указать, с какой он группы (если, конечно, вообще студент, что тоже не факт).
Через деканат - но по каждому не будешь спрашивать. Электронный журнал? Там нет магистрантов, да и сам поиск крайне корявый - и ищет среди всех студентов вуза, а не только наших подопечных.
В открытом доступе в Сети есть PDF-файлы с рейтингом студентов - включает данные о группе, ФИО и собственно рейтинге. Жаль, поиск в PDF, да еще в кучке файлов, тоже удобным не назовешь.
Поэтому стоит задача считать из PDF (ряда файлов), локально расположенных (т.е. их уже заранее скачал откуда следует), и записать данные о студентах в отдельную коллекцию, которую затем сериализовать в XML или хранить в БД. Смотря насколько лень будет возиться. Главное, что при повторном считывании можно будет сразу обратиться к такому файлу, и иметь дело с одной коллекцией, в которой не проблема ни поиск, ни фильтрация. 
Плюс легко сделать возможность по одному нажатию (или хоткею) заносить в буфер обмена требуемых именно для моих задач данных по найденному студенту - ФИО с группой и только группу.
Такая программка сильно упростит формирование отчетов, где фигурируют студенты с не-студентами.

Для ридми:
Программа для агрегирования информации о студентах, полученной из доступных для скачивания PDF-файлах с данными о стипендиальном рейтинге.
Считанная из файлов информация (ФИО, рейтинг, группа, примечания) помещается в список, который затем может быть сериализован в XML-файл для повторного использования. 
После открытия выбранного XML-файла заполняется таблица, вывод в которую можно фильтровать по мере ввода фамилии.
Программа позволяет формировать строковые заготовки с данными выбрраного студента (Фамилия И.О. (Группа), просто (Группа) и т.д.) для различных крайне важных отчетов.
The program for aggregating information about students, obtained from downloadable PDF-files with data on the scholarship rating.
The information retrieved from the files (name, rating, group, notes) is placed in the list, which can then be serialized into an XML file for reuse.
After opening the selected XML-file, the table is populated, the output to which can be filtered as the surname is entered.
The program allows you to create string blanks with the data of the selected student (Surname I.O. (Group), only (Group), etc.) for various extremely important reports.
